SuperTracker CTE Tracker Viewer

An interactive viewer for CTE tracker .elog files (pipe-delimited with repeating header blocks), built on top of the SuperCSV widget layer. Provides full filtering, sorting, column-hiding, email, and export capabilities — with native support for plain CSV / TSV / PSV files as well.

Overview

SuperTracker is a specialised viewer for CTE tracker .elog files. These files are pipe-delimited and contain a repeating 3-line header block scattered every ~50 rows throughout the file. SuperTracker strips all repeated headers and loads the data into the same FilteredTable widget used by SuperCSV.

Plain CSV / TSV / PSV files are also fully supported and are routed through SuperCSV's regular rendering path — extended with header-block-aware reading so multi-row or repeating headers in CSV files can be cleaned up identically to elog files.

Features

Elog parser

Auto-detects pipe-delimited tracker logs and strips repeated ===…=== separator + header blocks.

Gzip support

Reads .elog.gz directly — no manual decompression needed.

Multi-row headers

--header-row N:k handles header blocks spanning multiple lines.

Hex / binary range filter

Tracker tabs accept ranges like 0x10..0x2F and 0b1010..0b1111.

Multi-tab UI

Open many files at once, each in its own tab.

Filter / sort / hide

Per-column filter expressions, global logic bar, column picker — all inherited from SuperCSV.

Email & export

Send filtered data via email or export to CSV / Excel.

Recent files

Up to _RECENT_MAX recently-opened files persisted across sessions.

Dependencies

SuperTracker builds on SuperCSV's shared widget layer:

  • FilteredTable — the data grid widget
  • FontManager — global font scaling
  • ThemeManager — light/dark themes (used by inherited toolbar)
  • SuperCSV — the base application class

The bin/supertracker wrapper script automatically prepends tools/supercsv/ to PYTHONPATH, so no manual setup is needed.

External: pandas>=1.3 (matches SuperCSV's requirement).

Quick Start

bin/supertracker path/to/file.elog.gz
bin/supertracker                          # opens file-picker dialog
bin/supertracker trk_T0.elog.gz trk_T1.elog.gz   # multiple files / tabs

Elog Format

CTE tracker logs contain a repeating 3-line header pattern interleaved with data rows:

====================...==     ← separator (≥ 20 '=' characters)
col1 | col2 | col3             ← column-name line
====================...==     ← separator
row1a | row1b | row1c          ← data rows (repeats every ~50 rows)
row2a | row2b | row2c
…
====================...==     ← next header block — stripped
col1 | col2 | col3
====================...==
…

SuperTracker's parser uses a small state machine to detect each header occurrence and remove it, keeping only the first set of column names. See _parse_elog_stream() in supertracker.py.

Header-Row Spec (--header-row)

The header-row specification controls how non-tracker delimited files are parsed:

SpecMeaning
0Single-row header at row 0 (default)
NSingle-row header at 0-based row N
N:kHeader block of k rows starting at row N; column names taken from the last non-separator row of the block
Repeated headers: Every row in the file that exactly matches any row of the header block is stripped from the data — so periodic re-printed headers are removed automatically.

Hex / Binary Range Filtering

Elog tabs install an extended RANGE filter that understands numeric prefixes in both the filter spec and the cell values:

  • Decimal: 10..20
  • Hex: 0x10..0x2F
  • Binary: 0b1010..0b1111
  • Mixed: 0x10..255

Standard CSV tabs continue to use SuperCSV's decimal-only range behaviour. The override is implemented via _xt_parse_number(), _xt_parse_range_pairs(), _xt_coerce_series() and _xt_apply_col_filter().

Architecture

SuperTracker is a thin extension of SuperCSV:

SuperCSV (tools/supercsv/supercsv.py)
   └── SuperTracker (tools/supertracker/supertracker.py)
        • overrides _open_dialog()  → adds elog file types
        • overrides _open_csv()     → routes elog files to _load_elog_as_df()
        • adds _open_elog()         → tracker-tab loader with hex/bin range filter
        • adds _ask_tracker_csv_options() → extended dialog for non-tracker files

All UI surfaces — tabs, toolbar, filter bar, column picker, font controls, recent-files menu, export / email actions — are inherited unchanged from SuperCSV.

Examples

Open a single tracker file

python3 supertracker.py fe_ifu_ic_sb_trk_T0.elog.gz

Open several tracker logs

python3 supertracker.py trk_T0.elog.gz trk_T1.elog.gz

Mix tracker and CSV files

python3 supertracker.py results.elog.gz summary.csv

Space-delimited file, 3-row header starting at row 2, 8 columns

python3 supertracker.py -d ' ' --header-row 2:3 --num-cols 8 my_report.txt

Drop non-compliant rows instead of padding

python3 supertracker.py --drop-bad-lines data.csv

Open file dialog (no args)

python3 supertracker.py

Tests

Unit tests live in tests/test_supertracker.py and cover the elog parser, header-spec parsing, separator detection, hex / binary range filters, and the tracker CSV loader.

cd tools/supertracker
python3 -m pytest tests/
