💡 Core concepts

A few small ideas that the rest of the product is built on.

Markdown is canonical

Every task lives inside a .md file on disk. The SQLite index is rebuilt from those files; you can edit them in any editor and the UI catches up.

Tokens, not fields

!task, #eta, #owner, #priority are inline annotations. The parser is a pure function: parse(md) → {tasks, attrs, refs}.

Stable IDs (#id T-…)

The agenda-roll button injects a 6-character Crockford id into every task. Keep it — references and links resolve through it.

Reference rows

A bullet starting with #task T-… is not a new task — it points at the canonical one and any tokens on that line are write-through overrides.

Solo gamification

Stats & badges are private to you. No leaderboards, no comparisons, no manager dashboards. Recordings are server-side; surfacing is per-client.

One auth model

HTTP Basic everywhere. The CLI re-uses ~/.veganotes/credentials; the web reuses the browser-cached header. Profiles let you point at multiple servers.

✍️ Markdown syntax

A small grammar that turns plain markdown into structured work. Unknown tokens are preserved verbatim.

Token reference

TokenMeaningExample
!task <title>Declares a task. Title runs until the next known #token.- !task Wire up auth #eta +2d
!AR <title>Action-required item. Same shape, kind ar.- !AR follow up with vendor
#task <ID>Reference to an existing task. Leading-position = agenda reference row; mid-prose = directed link.- #task T-A3F9K2 #status done
#id <ID>Stable identifier auto-injected by the agenda roll (Crockford-base32, 6 chars).!task Wire SSO #id T-A3F9K2
#eta <when>Due date. ISO, +2d, tomorrow, next fri, or wwNN[.D].#eta ww18.4
#priority <val>Free user vocabulary (P0..P3, high/med/low).#priority P1
#project <name>Project bucket.#project veganotes
#owner <name>Owner. Repeat for multiple.#owner alice #owner bob
#status <val>Free-form. Trigger words bucket it into todo/in-progress/blocked/done for the Kanban.#status blocked by HSD approval
#estimate <dur>Effort: 30m, 2h, 1d, 0.5w.#estimate 4h
#feature <name>Cross-cutting feature label.#feature search-rewrite
#link <slug>Generic bidirectional link.#link rfc-2026-04

Rules of thumb

  • A token attaches to the nearest enclosing !task — same line or the most recent !task above in the same list block.
  • Indentation is 2 spaces (or one tab = 4). Children become subtasks of their parent.
  • A blank line ends the current task block.
  • Unknown #foo bar tokens are preserved verbatim and round-trip on re-serialize.
  • Reference rows (- #task T-… as the first token) are write-through overrides, not new tasks.

Live token playground

Paste markdown — the parser highlights tokens it recognises.

Pro tip. Status accepts free-form text. Write #status waiting on PSW review and the Kanban still buckets it as blocked while keeping your prose on disk.

✅ Tasks & ARs

The lifecycle of a unit of work, end-to-end.

Create

Three equally-good ways:

  • Inline in a note — type - !task Do the thing #owner alice #eta +3d anywhere in a list.
  • From the web — My Tasks, Kanban, and the editor each expose a "+ New" composer that stamps the right tokens for you.
  • From the CLI — vn note new for a fresh note, or use a vn api POST /api/tasks escape hatch.

Update

  • Drag in Kanban — drop the card in a new column and the underlying markdown is rewritten.
  • Click in Agenda / My Tasks — popover edits status, owner, priority, eta.
  • Reference row — in any note, write - #task T-A3F9K2 #status done and the indexer applies the override on save.
  • vn task — vn task T-A3F9K2 status=done priority=P1 eta=2026-W18.

Link tasks

  • Drop #task T-… mid-sentence to create a directional link.
  • Use #link <slug> for generic many-to-many cross-references.
  • The Graph view renders both as a navigable network.

Action Requests (ARs)

!AR is identical to !task with kind ar. Use it when an item is "someone else needs to do this" — the Kanban filters and AR-only badges treat it as a separate stream. Add them inline or with vn api POST /api/tasks/T-…/ars.

Anatomy of a worked example

# Sprint 14

- !task Wire up SSO #project veganotes #owner alice #eta +5d #priority P1
  - !task Add login screen #owner bob #eta +6d
  - !task Add OAuth callback #owner alice #eta +7d #status in-progress
- !task Migrate index #feature search-rewrite #status done
  Blocked by #task T-A3F9K2 and tracked in #link rfc-2026-04

🔎 Filtering & DSL

A tiny query language that powers the chip bar and the vn list -w flag.

Fixed columns

Every view supports owner, project, feature, priority, status, hide_done, and a free-text q search.

Where chips

Click + chip in the filter bar (or pass -w to vn list) to compose richer predicates:

ClauseEffect
owner=aliceTasks owned by alice.
@area=fit-valMatch the free-form attribute area with value fit-val.
eta>=ww18Range filter; works with ISO, ww, +Nd.
kind=arOnly ARs.
status!=doneNegation.
title~migrateSubstring match (case-insensitive).
Compose multiple chips. They AND together. The same string works in the URL bar — bookmarkable views are basically free.

Saved views

The + button next to the chip bar saves the current chip set as a named view. They're per-user and synced (GET /api/me/views / PUT /api/me/views). The command palette (⌘/Ctrl+K) jumps to any saved view.

🏆 Me & gamification

Solo, private, opt-out. Designed to be a small dopamine hit, never a comparison engine.

What's on the page

  • Stats card — closes today / 7d / 30d / lifetime, notes touched, on-time-ETA rate, favorite project.
  • Streak card — current + longest streak in days, plus rest tokens (you get 2 per rolling 14-day window — one off day doesn't kill a long run).
  • Badge grid — earned tiles (amber), locked tiles with progress bars, and a +N hidden footer for badges you'll discover by playing.
  • History sparkline — closes vs. note edits per day with 7 / 30 / 90 day toggle.
  • Recent activity — paginated event log, filterable by kind.
  • TZ widget — your IANA timezone. Streaks roll over at your midnight, not UTC.

🏆 First Lightclose your first task
🏆 Hat Trickclose 3 tasks in one day

Celebrations

When a write crosses a badge threshold, the API echoes awarded_badges: [...] and the UI pops a 🏆 toast in the corner. The CLI prints unlocked: <Name> — <blurb> after the normal output.

Opt-out

Hide it whenever you want.
  • Web: the show gamification toggle at the top of the Me page (stored in localStorage).
  • CLI: vn config gamify=off. Re-enable with vn config gamify=on.
  • The server keeps recording either way, so flipping back on restores your full history.
  • vn me tz stays available even with gamification off — TZ is a generic preference.

📦 CLI — install & auth

vn is a stdlib-only Python helper. No third-party deps, runs on Python ≥ 3.9.

One-liner install

/path/to/VegaNotes/tools/vn/install.sh
# Options: --no-editable | --system | --python python3.11 | --uninstall

Or run pip yourself from the repo root:

python3 -m pip install --user -e VegaNotes/tools/vn
# puts a `vn` shim in ~/.local/bin (drop --user inside a venv)

Authenticate

Lookup order, first hit wins:

  1. Env vars — VEGANOTES_URL, VEGANOTES_USER, VEGANOTES_PASS.
  2. ~/.veganotes/credentials (INI):
# ~/.veganotes/credentials
[default]
url = http://localhost:8000
user = alice
password = s3cret

[prod]
url = https://veganotes.example.com
user = alice
password = otherone

Pick a profile other than [default] with --profile prod or VEGANOTES_PROFILE=prod. chmod 600 the file.

⚙️ CLI — vn config

Per-profile client-side settings. Stored in ~/.veganotes/config (INI).

KeyDefaultEffect when off
gamifyonHide all vn me <subcommand> output and suppress badge celebrations.
gamify.notifyonKeep stats/badges working, but suppress the unlocked: celebration line.
 
vn config                       # list keys, values, and which are defaults
vn config gamify                # show one key
vn config gamify=off            # key=value form
vn config gamify.notify off     # space-separated form
Config flips don't change what the server records. Turn gamification off for a sprint, back on later, your full history is intact.

🔭 CLI — vn show

Read-only inspector. Every resource respects --format {table,json,jsonl,csv,ids} and --columns col1,col2,….

vn show projects — list projects you can see

Each row: name, role (manager/member). Pass a project name for member list and notes.

vn show projects
vn show projects ww18
vn show projects --format json | jq
vn show tasks — filter the task list

Same flags as vn list. Add columns:

vn show tasks --owner alice --status open
vn show tasks --columns +kind-status     # add kind, drop status
vn show tasks -w '@area=fit-val' -w 'eta>=ww18'
vn show task T-… — one task

Use --format json to get every field including links and notes:

vn show task T-A3F9K2
vn show task T-A3F9K2 --format json | jq .links
vn show notes — browse the markdown corpus 
vn show notes                      # id / path / etag
vn show notes 42                   # one note (header)
vn show notes 42 --full            # include body
vn show notes projects/ww18/foo.md # path also works
vn show tree — project → note → task tree

Indented map of every project you can see, every note in it, and every task on those notes.

vn show tree
vn show agenda — next-N-days view 
vn show agenda --owner alice --days 14
vn show links T-… — inbound & outbound references 
vn show links T-A3F9K2
vn show search 'query' — cross-resource search

Full-text search over notes and task titles. Returns rows tagged with their resource type.

vn show search 'fit-val'
vn show search 'sso' --format json
vn show me — self info + saved views 
vn show me

📌 CLI — tasks & notes

Everyday mutations from the shell.

Mutate a task

vn task T-A3F9K2 status=done
vn task T-A3F9K2 status=done priority=P1 eta=2026-W18
vn task T-A3F9K2 add_note='blocked on PSW review'

Field shorthands: status=, priority=, eta=, owners=alice,bob, features=auth,sso, add_note=… (appends), notes=… (replaces).

List tasks

vn list --owner alice --status open
vn list -w '@area=fit-val' -w 'eta>=ww18' --sort eta:desc
vn list --columns id,title,status,eta,owner --format csv

New note

vn note new --project ww18 --title 'standup notes'
vn note new --project ww18 --title 'design review' --open  # opens $EDITOR

Identity check

vn whoami
Output formats apply everywhere. --format {table,json,jsonl,csv,ids} works for vn list, vn show *, and vn me *. ids is great for piping into another vn task.

🎯 CLI — vn me

Solo gamification surfaces. All reads, all private.

vn me stats — full stats card 
vn me stats
vn --json me stats | jq .current_streak_days
vn me streak — headline streak only 
vn me streak
vn me history --days 30 — ANSI sparkline of closes/edits per day 
vn me history --days 7
vn me history --days 90
vn me activity — recent event log 
vn me activity
vn me activity --kind task.closed --limit 20
vn me activity --since 2026-04-01 --until 2026-04-15
vn me badges — earned achievements 
vn me badges            # earned only
vn me badges --all      # also list locked + progress
vn me tz — show / set IANA timezone always on

Exempt from gamify=off because TZ is a general preference.

vn me tz                            # show
vn me tz America/Los_Angeles        # set (streaks roll local)
vn me tz UTC                        # reset

🛠️ CLI — vn api

Generic HTTP escape hatch for endpoints vn show doesn't cover, prototyping, or admin routes.

vn api GET   /api/admin/users
vn api GET   /api/tasks --query 'project=ww18' --query 'kind=ar'
vn api POST  /api/projects --json-body '{"name":"ww19"}'
vn api PATCH /api/tasks/T-A3F9K2 --json-body '{"status":"done"}'

Output formats: --format json (default), jsonl, raw. HTTP errors print HTTP <status>: <body> to stderr and exit 4 (4xx) or 5 (5xx).

The API is documented under /docs on any running server (FastAPI auto-spec). When in doubt, hit vn api GET /openapi.json | jq .paths.

📚 Recipes

Battle-tested patterns the team uses every week.

Daily stand-up

vn show agenda --owner $USER --days 1     # what's due today
vn me streak                              # quick morale check
vn me activity --since $(date -d yesterday +%F)  # what you closed

Weekly roll

Hit the roll button in the editor (or vn api POST /api/notes/next-week) to copy this week's note forward, stamping any missing #ids.

Triage someone else's queue

vn show tasks --owner alice --status open --columns id,title,eta,priority --format table

Bulk-close stale items

vn list -w 'eta<ww17' --status open --format ids \
  | xargs -I{} vn task {} status='dropped — sprint over'

Audit your closes for a sprint

vn me activity --kind task.closed --since 2026-04-14 --until 2026-04-28 --format csv

Onboard a new project

vn api POST /api/projects --json-body '{"name":"ww19"}'
vn api PUT  /api/projects/ww19/members --json-body '{"user_name":"alice","role":"manager"}'
